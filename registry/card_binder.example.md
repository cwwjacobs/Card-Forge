# Example Card Binder

A Card Binder is the human-readable view of a Card Collection.

The source of truth remains `registry/card_registry.jsonl`.

## Universal / Basic Cards

### Source Inventory

- Card ID: `check.source_inventory.v0_1`
- Type: Check Card
- Use: Start a Run by listing and classifying source material.

### No Overclaim

- Card ID: `contract.no_overclaim.v0_1`
- Type: Contract Card
- Use: Prevent unsupported readiness, safety, compliance, or production claims.

### Human Approval Gate

- Card ID: `gate.human_approval.v0_1`
- Type: Gate Card
- Use: Pause before risky actions, index changes, publishing, spending, or external communications.

## Binder rule

The Binder helps humans browse Cards. The Registry remains the source of truth.
